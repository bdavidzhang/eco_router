# DGX Spark Sensor Discovery Toolkit 🌡️

Tools to discover and monitor hardware sensors on the NVIDIA DGX Spark (ARM64/aarch64).

## Quick Start

```bash
# 1. Make scripts executable
chmod +x *.sh

# 2. Run full sensor discovery (does everything)
bash discover_sensors.sh

# 3. Or run continuous monitor
bash monitor.sh

# 4. Or collect everything for analysis
bash collect_all.sh
# Output will be in sensor_data_YYYYMMDD_HHMMSS/
```

## What's Included

| Script | Purpose | Time |
|--------|---------|------|
| `discover_sensors.sh` | **Full system probe** - runs everything | ~30s |
| `nvidia_sensors.sh` | NVIDIA GPU/tegra stats (nvidia-smi, tegrastats) | ~5s |
| `thermal_zones.sh` | Linux thermal zones (/sys/class/thermal) | ~3s |
| `hwmon_sensors.sh` | Hardware monitoring (/sys/class/hwmon) | ~3s |
| `ipmi_sensors.sh` | IPMI/BMC sensors (if available) | ~5s |
| `lm_sensors.sh` | lm-sensors setup and readings | ~3s |
| `collect_all.sh` | **Collects everything + raw sysfs dumps** | ~1min |
| `monitor.sh` | **Live dashboard** (like `watch`) | continuous |

## System Info

- **Device**: NVIDIA DGX Spark
- **Architecture**: ARM64 (aarch64)
- **OS**: Ubuntu GNU/Linux 6.17.0-1008-nvidia
- **Expected Sensors**: CPU, GPU, DRAM, NVMe, PMIC, thermal zones

## Installing Dependencies

Some scripts need extra packages. Install them all at once:

```bash
sudo apt update
sudo apt install -y lm-sensors ipmitool nvme-cli smartmontools bc

# Configure lm-sensors (optional, follow prompts)
sudo sensors-detect
```

## Usage Examples

### Quick discovery
```bash
bash discover_sensors.sh | less
```

### Just GPU temps
```bash
bash nvidia_sensors.sh | grep -i temp
```

### Watch temps live
```bash
bash monitor.sh  # Updates every 2 seconds
bash monitor.sh 5  # Updates every 5 seconds
```

### Full data collection for sharing
```bash
bash collect_all.sh
# Creates sensor_data_YYYYMMDD_HHMMSS/ with all data
```

### Check specific sensor type
```bash
bash thermal_zones.sh | grep -i gpu
bash hwmon_sensors.sh | grep -i power
```

## Troubleshooting

### "Permission denied" on sensor files
Some sensors need root:
```bash
sudo bash discover_sensors.sh
```

### Missing commands
Install the packages listed above. The scripts will tell you what's missing.

### No GPU sensors
Check if nvidia-smi works:
```bash
nvidia-smi
# If not found, you may need NVIDIA drivers
```

### Monitor looks weird
The monitor uses ANSI colors. If your terminal doesn't support them, edit monitor.sh and set `CLEAR_SCREEN=false`.

## What to Look For

On DGX Spark, you might find sensors for:
- **CPU cores** (thermal zones)
- **GPU** (nvidia-smi, tegrastats, or thermal zones)
- **DRAM/Memory controller** (hwmon or PMIC)
- **NVMe SSD** (nvme-cli or smartctl)
- **PMIC** (power management IC - usually in hwmon)
- **System/Board** temperature
